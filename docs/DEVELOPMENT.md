## 快速开始

### 什么是拓展包？

拓展包是一个 JSON 文件，包含自定义的游戏数据，可以扩展 Nexus 命令助手的功能。通过拓展包，你可以：

- ✅ 添加自定义方块 ID 与翻译
- ✅ 添加自定义物品 ID 与翻译
- ✅ 添加自定义音效 ID 与翻译
- ✅ 添加自定义粒子 ID 与翻译
- ✅ 添加快捷命令
- ✅ 添加命令模板

### 创建你的第一个拓展包

1. 复制 [示例模板](../examples/template.json)
2. 用文本编辑器打开
3. 修改 `id`、`name`、`author` 等基本信息
4. 添加你的自定义内容
5. 保存为 `.json` 文件
6. 在 Nexus 中导入使用

### 最小拓展包

```json
{
  "id": "your_name.my_first_addon",
  "name": "我的第一个拓展包",
  "version": "1.0.0",
  "author": "你的名字"
}
```

## 开发工作流

### 1. 规划拓展包内容

在开始之前，先规划好你的拓展包：

- **主题**: 这个拓展包是关于什么的？（例如：某服务器的自定义物品、红石工具包、建筑模板等）
- **范围**: 包含哪些类型的内容？（方块/物品/音效/粒子/命令/模板）
- **命名空间**: 使用什么命名空间？（建议使用个人或团队名）

### 2. 编写 JSON

使用你喜欢的文本编辑器编写 JSON。推荐使用支持 JSON 语法高亮和验证的编辑器：

- VS Code
- Sublime Text
- 在线工具: [jsonlint.com](https://jsonlint.com/)

### 3. 验证格式

使用以下方法验证你的拓展包格式是否正确：

**方法一：在线验证**
1. 访问 [jsonlint.com](https://jsonlint.com/)
2. 粘贴你的 JSON 内容
3. 点击验证

**方法二：使用 JSON Schema**
使用 [schema/addon.schema.json](../schema/addon.schema.json) 进行更严格的验证。

**方法三：在 Nexus 中导入测试**
直接在 Nexus 拓展包管理页面导入，查看是否导入成功。

### 4. 在 Nexus 中测试

1. 打开 Nexus 命令助手
2. 进入「设置」→「拓展包管理」
3. 点击右上角「+」按钮
4. 粘贴你的拓展包 JSON 内容
5. 点击「导入」
6. 检查各功能页是否正确显示新增内容

### 5. 迭代优化

根据测试结果调整拓展包内容：
- 修正拼写错误
- 调整分类
- 完善描述
- 添加更多内容

## 最佳实践

### ✅ ID 命名

- **使用独特的命名空间**: 例如 `com.yourname.addon_name`
- **全小写**: 所有 ID 使用小写字母和下划线
- **见名知意**: ID 应该能反映内容的含义
- **不要使用 `minecraft:` 前缀**: 这是原版命名空间，可能导致混淆

```json
// ✅ 好的命名
"id": "custom:ruby_sword"

// ❌ 不好的命名
"id": "minecraft:ruby_sword"  // 使用了原版命名空间
"id": "RubySword"             // 包含大写字母
"id": "rs"                    // 太短，含义不清
```

### ✅ 命名与翻译

- **中文名称准确**: 使用玩家常用的中文译名
- **描述简洁明了**: 一句话说清楚是什么
- **分类一致**: 相似的内容放在同一分类

### ✅ 命令编写

- **以 `/` 开头**: 所有命令必须以斜杠开头
- **使用相对坐标**: 优先使用 `~` 和 `^` 相对坐标
- **使用目标选择器**: 优先使用 `@s`、`@p` 等选择器
- **参数完整**: 确保命令语法正确，可以直接使用

```json
// ✅ 好的命令
"command": "/effect @s speed 60 2 true"

// ❌ 不好的命令
"command": "effect Steve speed 60 2 true"  // 没有 /，使用了具体玩家名
```

### ✅ 版本管理

使用 [语义化版本](https://semver.org/lang/zh-CN/)：

- **主版本号 (MAJOR)**: 不兼容的 API 变更
- **次版本号 (MINOR)**: 向下兼容的功能性新增
- **修订号 (PATCH)**: 向下兼容的问题修正

```
1.0.0 → 1.0.1  # 修复错误
1.0.1 → 1.1.0  # 新增内容
1.1.0 → 2.0.0  # 重大变更
```

### ✅ 文件大小

- 单个拓展包建议不超过 1MB
- 内容较多时考虑拆分为多个拓展包

### ❌ 禁止内容

- 不要包含恶意代码或指令
- 不要侵犯他人版权
- 不要使用冒犯性或不当的名称
- 不要伪造原版内容（使用 `minecraft:` 命名空间）

## 调试与验证

### 常见格式错误

#### 1. 缺少逗号

```json
// ❌ 错误
{
  "id": "my_addon"
  "name": "我的拓展包"
}

// ✅ 正确
{
  "id": "my_addon",
  "name": "我的拓展包"
}
```

#### 2. 多余的逗号

```json
// ❌ 错误
{
  "id": "my_addon",
  "name": "我的拓展包",
}

// ✅ 正确
{
  "id": "my_addon",
  "name": "我的拓展包"
}
```

#### 3. 字符串未闭合

```json
// ❌ 错误
"name": "我的拓展包

// ✅ 正确
"name": "我的拓展包"
```

#### 4. 使用中文引号

```json
// ❌ 错误
"name": "我的拓展包"

// ✅ 正确
"name": "我的拓展包"
```

### 导入失败排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| JSON 格式无效 | 语法错误 | 使用 jsonlint.com 检查 |
| 缺少必填字段 | 缺少 id/name/version/author | 检查是否填写了所有必填字段 |
| ID 格式不正确 | 使用了大写或特殊字符 | 确保 ID 全小写，只含字母数字下划线 |
| 命令格式不正确 | 命令没有以 / 开头 | 确保所有 command 字段以 / 开头 |

## 发布与分享

### 准备发布

发布前请检查：

- [ ] 拓展包功能正常
- [ ] 所有 ID 命名规范
- [ ] 描述信息完整准确
- [ ] 版本号正确
- [ ] 没有侵权内容

### 分享方式

1. **直接分享 JSON 文件**
   - 将 JSON 文件发送给朋友
   - 对方在 Nexus 中导入即可

2. **发布到代码托管平台**
   - GitHub / Gitee 等
   - 方便版本管理和更新

3. **社区分享**
   - Minecraft 相关社区
   - QQ 群 / 论坛

### 推荐的 README 模板

如果你将拓展包发布到 GitHub，建议包含以下内容：

```markdown
# 拓展包名称

简短描述

## 功能特性

- 功能1
- 功能2

## 安装方法

1. 下载 `addon.json`
2. 打开 Nexus → 设置 → 拓展包管理
3. 点击导入，粘贴内容

## 内容统计

| 类型 | 数量 |
|------|------|
| 方块 | 0 |
| 物品 | 0 |
| 命令 | 0 |

## 版本历史

### v1.0.0
- 初始版本
```

## 常见问题

### Q: 拓展包最多能包含多少内容？

A: 没有硬性限制，但建议单个拓展包不超过 1MB。内容过多时建议拆分。

### Q: 可以修改原版内容吗？

A: 不能。拓展包只能添加新内容，不能修改或覆盖原版数据。

### Q: 多个拓展包的 ID 冲突怎么办？

A: 如果两个拓展包有相同的内容 ID，后安装的会覆盖先安装的。建议使用独特的命名空间避免冲突。

### Q: 如何卸载拓展包？

A: 在拓展包管理页面，点击拓展包进入详情，然后点击「卸载拓展包」。

### Q: 拓展包会影响游戏性能吗？

A: 不会。拓展包只是增加命令补全和命令库的数据，不会影响游戏运行。

### Q: 可以在服务器中使用吗？

A: 拓展包只是客户端的辅助工具，不影响服务器。但如果拓展包包含的是服务器自定义内容，需要对应服务器安装了相应的插件/模组才能生效。

### Q: 如何反馈 Bug 或提建议？

A: 可以在 Nexus 项目的 GitHub Issues 中反馈。
