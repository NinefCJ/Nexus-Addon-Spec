## 快速开始

### 什么是拓展包？

拓展包是一个 JSON 文件，包含自定义的游戏数据，可以扩展 Nexus 命令助手的功能。通过拓展包，你可以：

- ✅ 添加自定义方块 ID 与翻译
- ✅ 添加自定义物品 ID 与翻译
- ✅ 添加自定义音效 ID 与翻译
- ✅ 添加自定义粒子 ID 与翻译
- ✅ 添加自定义动画 ID 与翻译
- ✅ 添加快捷命令
- ✅ 添加命令模板
- ✅ 提供完整的命令库和枚举数据
- ✅ 提供方块状态定义
- ✅ 提供 JSON Schema 校验

### 拓展包类型一览

| 类型 | 难度 | 适用人群 | 核心功能 |
|------|------|----------|----------|
| 简易拓展包 (simple) | ⭐ 入门 | 普通玩家 | 添加自定义 ID、翻译、快捷命令 |
| 命令库拓展 (commandLib) | ⭐⭐⭐ 进阶 | 模组/服务器开发者 | 完整命令库、枚举数据 |
| 方块状态拓展 (blockStates) | ⭐⭐ 进阶 | 模组开发者 | 方块状态提示 |
| JSON Schema 拓展 (jsonSchema) | ⭐⭐⭐⭐ 高级 | 高级开发者 | NBT/JSON 智能提示 |
| 完整拓展包 (full) | ⭐⭐⭐⭐⭐ 专家 | 大型团队 | 全部功能的综合支持 |

### 5 分钟创建第一个拓展包

```json
{
  "id": "your_name.my_first_addon",
  "name": "我的第一个拓展包",
  "version": "1.0.0",
  "author": "你的名字",
  "description": "这是我的第一个 Nexus 拓展包",
  "items": [
    {
      "id": "custom:ruby_sword",
      "name": "红宝石剑",
      "category": "武器",
      "description": "一把由红宝石打造的神剑"
    }
  ],
  "commands": [
    {
      "id": "give_ruby_sword",
      "command": "/give @s custom:ruby_sword",
      "name": "获得红宝石剑",
      "description": "给予自己一把红宝石剑",
      "category": "道具"
    }
  ]
}
```

将以上内容保存为 `my-addon.json`，然后在 Nexus 中导入即可。

## 完整开发工作流

### 第一阶段：规划设计

### 1.1 确定拓展包类型和定位

在动手之前，先回答以下问题：

| 问题 | 说明 | 示例 |
|------|------|------|
| 目标用户是谁？ | 这个拓展包是给谁用的？ | 服务器玩家 / 红石玩家 / 建筑玩家 |
| 解决什么问题？ | 为什么需要这个拓展包？ | 服务器自定义物品补全 / 常用命令快捷输入 |
| 包含什么内容？ | 哪些类型的内容？ | 物品 + 命令模板 |
| 规模有多大？ | 大概多少条数据？ | 10-50 条 / 50-200 条 / 200+ |
| 选择什么类型？ | 简易/命令库/方块状态/完整 | 简易拓展包 |

**如何选择拓展包类型：**

| 你的需求 | 推荐类型 |
|----------|----------|
| 添加一些自定义物品/方块的翻译 | 简易拓展包 (simple) |
| 分享一些常用命令 | 简易拓展包 (simple) |
| 为模组添加完整的命令支持 | 命令库拓展 (commandLib) |
| 为自定义方块添加状态提示 | 方块状态拓展 (blockStates) |
| 为 NBT/JSON 添加智能提示 | JSON Schema 拓展 (jsonSchema) |
| 大型模组的完整 Nexus 适配 | 完整拓展包 (full) |

#### 1.2 设计命名空间

命名空间是拓展包的唯一标识，也是避免冲突的关键。

**命名规范：**
- 格式：`作者名.拓展包名` 或 `反向域名格式`
- 全小写，使用下划线分隔单词
- 不要使用 `minecraft` 作为命名空间

**推荐格式：**
```
个人作者：  yourname.addon_name
团队/服务器： teamname.server_addon
项目：      com.projectname.addon
```

**示例：**
```
✅ steve.ruby_items
✅ minecraft_server.custom_weapons
✅ com.example.building_templates

❌ RubyItems          // 包含大写
❌ my addon           // 包含空格
❌ minecraft.stuff    // 使用了原版命名空间
```

#### 1.3 内容规划表

建议先列一个内容清单：

| 类型 | 数量 | 分类 | 备注 |
|------|------|------|------|
| 方块 | 5 | 建筑/装饰 | 建筑用的装饰方块 |
| 物品 | 12 | 武器/工具/材料 | 自定义武器和材料 |
| 动画 | 8 | 攻击动作/表情动作 | 自定义动画 |
| 命令 | 8 | 常用/管理 | 服务器常用命令 |
| 模板 | 3 | 建筑/机械 | 大型命令模板 |

### 第二阶段：开发编写

#### 2.1 选择编辑器

推荐使用 **VS Code**，配合 JSON Schema 获得智能提示和实时校验。

> 详见 [VS Code 开发环境配置](#vs-code-开发环境配置)

#### 2.2 从模板开始

复制 [examples/template.json](../examples/template.json) 作为起点，按需修改。

#### 2.3 编写内容

按照 [格式规范](./SPEC.md) 中的字段说明，逐一添加内容。

**编写建议：**
- 写一点，保存一下，避免意外丢失
- 每添加 5-10 条就导入测试一次
- 保持分类的一致性

#### 2.4 版本号规范

使用 [语义化版本](https://semver.org/lang/zh-CN/) `主版本.次版本.修订号`：

| 版本号变化 | 场景 | 示例 |
|-----------|------|------|
| 修订号 +1 | 修复拼写错误、修正描述 | 1.0.0 → 1.0.1 |
| 次版本 +1 | 新增内容、新增分类 | 1.0.1 → 1.1.0 |
| 主版本 +1 | 大规模重构、删除旧内容 | 1.1.0 → 2.0.0 |

### 第三阶段：验证测试

#### 3.1 格式校验

**Step 1: JSON 语法检查**

使用 [jsonlint.com](https://jsonlint.com/) 或 VS Code 内置校验。

**Step 2: Schema 校验**

使用 [addon.schema.json](../schema/addon.schema.json) 进行完整校验。

**Step 3: 人工检查清单**

- [ ] 所有必填字段已填写
- [ ] ID 命名规范（全小写、下划线、独特命名空间）
- [ ] 没有使用 `minecraft:` 前缀的自定义内容
- [ ] 命令都以 `/` 开头
- [ ] 分类名称一致，没有拼写错误
- [ ] 描述简洁准确

#### 3.2 在 Nexus 中测试

**导入测试步骤：**

1. 打开 Nexus → 设置 → 拓展包管理
2. 点击「+」导入按钮
3. 粘贴 JSON 内容
4. 检查是否导入成功

**功能测试清单：**

| 测试项 | 测试方法 |
|--------|----------|
| 方块补全 | 输入 `/setblock ~ ~ ~ ` + 命名空间，检查是否出现 |
| 物品补全 | 输入 `/give @s ` + 命名空间，检查是否出现 |
| 音效补全 | 输入 `/playsound ` + 命名空间，检查是否出现 |
| 粒子补全 | 输入 `/particle ` + 命名空间，检查是否出现 |
| 快捷命令 | 检查快捷命令面板是否显示 |
| 命令模板 | 检查命令库中是否有拓展包分类 |
| 禁用/启用 | 切换开关，确认内容正确显示/隐藏 |
| 卸载 | 卸载后确认内容完全移除 |

> 详细的调试方法请参考 [调试与测试指南](./DEBUGGING.md)

## 最佳实践

### ✅ ID 命名最佳实践

- **使用独特的命名空间**: 例如 `com.yourname.addon_name`
- **全小写**: 所有 ID 使用小写字母和下划线
- **见名知意**: ID 应该能反映内容的含义
- **不要使用 `minecraft:` 前缀**: 这是原版命名空间，可能导致混淆

```json
// ✅ 好的命名
"id": "custom:ruby_sword"

// ❌ 不好的命名
"id": "minecraft:ruby_sword"  // 使用了原版命名空间
"id": "RubySword"             // 包含大写字母
"id": "rs"                    // 太短，含义不清
```

### ✅ 命名与翻译最佳实践

- **中文名称准确**: 使用玩家常用的中文译名
- **描述简洁明了**: 一句话说清楚是什么
- **分类一致**: 相似的内容放在同一分类

**推荐的分类名称：**

| 类型 | 推荐分类 |
|------|----------|
| 方块 | 建筑、装饰、红石、自然、功能 |
| 物品 | 武器、工具、材料、食物、装备、杂项 |
| 音效 | 环境、生物、玩家、方块、音乐、UI |
| 粒子 | 环境、效果、魔法、红石 |
| 命令 | 常用、传送、管理、道具、建筑、娱乐 |
| 模板 | 建筑、机械、电路、系统 |

### ✅ 命令编写最佳实践

- **以 `/` 开头**: 所有命令必须以斜杠开头
- **使用相对坐标**: 优先使用 `~` 和 `^` 相对坐标
- **使用目标选择器**: 优先使用 `@s`、`@p` 等选择器
- **参数完整**: 确保命令语法正确，可以直接使用

```json
// ✅ 好的命令
"command": "/effect @s speed 60 2 true"

// ❌ 不好的命令
"command": "effect Steve speed 60 2 true"  // 没有 /，使用了具体玩家名
```

### ✅ 文件组织最佳实践

- **单主题原则**: 一个拓展包只做一件事，不要把不相关的内容混在一起
- **大小适中**: 单个拓展包建议 50-500 条内容，过多考虑拆分
- **分类清晰**: 合理使用 category 字段，便于用户浏览

### ❌ 禁止内容

- 不要包含恶意代码或指令
- 不要侵犯他人版权
- 不要使用冒犯性或不当的名称
- 不要伪造原版内容（使用 `minecraft:` 命名空间）

## VS Code 开发环境配置

### 1. 安装 VS Code

下载地址：[code.visualstudio.com](https://code.visualstudio.com/)

### 2. 配置 JSON Schema

在你的拓展包项目根目录创建 `.vscode/settings.json`：

```json
{
  "json.schemas": [
    {
      "fileMatch": ["*.addon.json", "addon.json"],
      "url": "https://raw.githubusercontent.com/NinefCJ/Nexus-Addon-Spec/main/schema/addon.schema.json"
    }
  ]
}
```

或者在 JSON 文件顶部添加 `$schema` 字段：

```json
{
  "$schema": "https://raw.githubusercontent.com/NinefCJ/Nexus-Addon-Spec/main/schema/addon.schema.json",
  "id": "your.addon",
  ...
}
```

### 3. 推荐插件

| 插件 | 用途 |
|------|------|
| JSON Tools | JSON 格式化、压缩 |
| Error Lens | 行内显示错误 |
| Todo Tree | 管理 TODO 注释 |

### 4. 高效编辑技巧

- **格式化**: `Shift + Alt + F` 快速格式化 JSON
- **折叠**: 点击行号旁的箭头折叠代码块
- **多光标**: `Alt + 点击` 添加多个光标，批量编辑
- **查找替换**: `Ctrl + H` 批量替换文本

## 常见问题

### Q: 拓展包最多能包含多少内容？

A: 没有硬性限制，但建议单个拓展包不超过 1MB。内容过多时建议按主题拆分。

### Q: 可以修改原版内容吗？

A: 不能。拓展包只能添加新内容，不能修改或覆盖原版数据。

### Q: 多个拓展包的 ID 冲突怎么办？

A: 如果两个拓展包有相同的内容 ID，后安装的会覆盖先安装的。建议使用独特的命名空间避免冲突。

### Q: 拓展包支持图片资源吗？

A: 目前不支持。拓展包是纯文本 JSON 文件，所有信息通过文字描述。

### Q: 如何卸载拓展包？

A: 在拓展包管理页面，点击拓展包进入详情，然后点击「卸载拓展包」。

### Q: 拓展包会影响游戏性能吗？

A: 不会。拓展包只是增加命令补全和命令库的数据，不会影响游戏运行。

### Q: 可以在服务器中使用吗？

A: 拓展包只是客户端的辅助工具，不影响服务器。但如果拓展包包含的是服务器自定义内容，需要对应服务器安装了相应的插件/模组才能在游戏中生效。

### Q: 如何反馈 Bug 或提建议？

A: 可以在 Nexus 项目的 GitHub Issues 中反馈。
