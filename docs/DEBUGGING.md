## 问题排查流程

当拓展包出现问题时，按照以下流程逐步排查：

```
发现问题
   ↓
是导入失败吗？ ──是──▶  检查 JSON 格式
   ↓否                           ↓
内容显示不正确？ ──是──▶  检查字段名称和类型
   ↓否                           ↓
功能不正常？ ──是──▶  检查 ID 和命令格式
   ↓否                           ↓
性能问题？ ──是──▶  检查数据量和重复项
   ↓否                           ↓
提交 Issue 反馈
```

## 常见格式错误

### 1. 缺少逗号

**错误示例：**
```json
{
  "id": "my_addon"
  "name": "我的拓展包"
}
```

**正确示例：**
```json
{
  "id": "my_addon",
  "name": "我的拓展包"
}
```

**排查方法：**
- 使用 VS Code 的语法高亮，错误位置会有红色波浪线
- 使用 jsonlint.com 粘贴检查，会指出具体行号

### 2. 多余的逗号

**错误示例：**
```json
{
  "id": "my_addon",
  "name": "我的拓展包",
}
```

**正确示例：**
```json
{
  "id": "my_addon",
  "name": "我的拓展包"
}
```

**注意：** JSON 不允许数组或对象最后一个元素后面有逗号。

### 3. 字符串未闭合

**错误示例：**
```json
"name": "我的拓展包
```

**正确示例：**
```json
"name": "我的拓展包"
```

### 4. 使用中文引号

**错误示例：**
```json
"name": "我的拓展包"
```

**正确示例：**
```json
"name": "我的拓展包"
```

**排查方法：** 仔细检查引号，确保都是英文双引号 `"`。

### 5. 字段名拼写错误

**错误示例：**
```json
{
  "items": [
    {
      "id": "custom:sword",
      "Name": "宝剑"
    }
  ]
}
```

**问题：** `Name` 大写了，应该是 `name`。

**正确示例：**
```json
{
  "items": [
    {
      "id": "custom:sword",
      "name": "宝剑"
    }
  ]
}
```

> 💡 **提示**: 使用 JSON Schema 可以自动检测字段名拼写错误。

### 6. 类型错误

**错误示例：**
```json
"version": 100
```

**问题：** `version` 应该是字符串，不是数字。

**正确示例：**
```json
"version": "1.0.0"
```

## 导入失败排查

| 错误提示 | 可能原因 | 解决方案 |
|---------|----------|----------|
| JSON 格式无效 | 语法错误（缺逗号、引号不闭合等） | 使用 jsonlint.com 检查具体错误位置 |
| 缺少必填字段 | 缺少 `id` / `name` / `version` / `author` | 对照 [格式规范](./SPEC.md) 检查必填字段 |
| ID 格式不正确 | ID 包含大写字母、空格或特殊字符 | 确保 ID 全小写，只含字母数字下划线和冒号 |
| ID 已存在 | 与已安装的拓展包 ID 冲突 | 修改 ID，使用独特的命名空间 |
| 命令格式不正确 | 命令没有以 `/` 开头 | 确保所有 `command` 字段以 `/` 开头 |
| 文件过大 | 文件超过大小限制 | 拆分内容为多个拓展包 |

### 逐步排查法

如果不确定问题出在哪里，可以用二分法快速定位：

1. 复制一份 JSON 文件作为备份
2. 删除一半的内容
3. 尝试导入
   - 如果成功 → 问题在被删除的那一半
   - 如果失败 → 问题在剩下的这一半
4. 继续缩小范围，直到找到问题

## 功能测试指南

### 测试清单

发布前请逐项检查以下内容：

#### 📦 基本信息

- [ ] 拓展包名称正确显示
- [ ] 作者名称正确显示
- [ ] 版本号正确显示
- [ ] 描述文字正确显示
- [ ] 图标（如果有）正确显示

#### 🧱 方块补全

测试命令：`/setblock ~ ~ ~ ` + 关键词

- [ ] 输入拓展包命名空间，能看到自定义方块
- [ ] 方块名称显示正确的中文翻译
- [ ] 点击补全能正确插入 ID
- [ ] 搜索关键词能正确匹配（中文名/英文名）

**测试用例：**
```
输入：/setblock ~ ~ ~ custom:
预期：显示所有 custom 命名空间的方块

输入：/setblock ~ ~ ~ 红宝石
预期：搜索到红宝石方块（如果中文名包含）
```

#### ⚔️ 物品补全

测试命令：`/give @s ` + 关键词

- [ ] 输入拓展包命名空间，能看到自定义物品
- [ ] 物品名称显示正确的中文翻译
- [ ] 点击补全能正确插入 ID
- [ ] 搜索关键词能正确匹配

#### 🔊 音效补全

测试命令：`/playsound ` + 关键词

- [ ] 输入拓展包命名空间，能看到自定义音效
- [ ] 音效名称显示正确的中文翻译
- [ ] 点击补全能正确插入 ID

#### ✨ 粒子补全

测试命令：`/particle ` + 关键词

- [ ] 输入拓展包命名空间，能看到自定义粒子
- [ ] 粒子名称显示正确的中文翻译
- [ ] 点击补全能正确插入 ID

#### ⚡ 快捷命令

- [ ] 快捷命令面板显示拓展包命令
- [ ] 点击命令能正确插入到输入框
- [ ] 命令名称和描述正确显示

#### 📋 命令模板

- [ ] 命令库中能找到拓展包分类
- [ ] 模板列表显示正确
- [ ] 点击模板能正确插入

#### 🔄 启用/禁用

- [ ] 禁用拓展包后，所有内容不再显示
- [ ] 重新启用后，内容恢复显示
- [ ] 切换状态不需要重启应用

#### 🗑️ 卸载

- [ ] 卸载后拓展包从列表中消失
- [ ] 所有相关内容完全移除
- [ ] 重新导入能正常工作

## 性能测试

### 数据量测试

| 数据量 | 预期表现 |
|--------|----------|
| < 50 条 | 秒开，无延迟 |
| 50-200 条 | 轻微延迟，可接受 |
| 200-500 条 | 明显延迟，建议拆分 |
| > 500 条 | 可能卡顿，必须拆分 |

### 性能优化建议

1. **避免重复数据**
   - 检查是否有重复的 ID
   - 同一条内容不要出现多次

2. **合理的描述长度**
   - 描述尽量简洁，一句话说明白
   - 不要在描述中放大段文字

3. **分类清晰**
   - 合理使用 category 字段
   - 分类数量适中（建议 3-10 个）

## 兼容性测试

### 与其他拓展包共存

- [ ] 安装多个拓展包后都能正常工作
- [ ] 不同拓展包的内容不会互相干扰
- [ ] 禁用一个不影响其他的

### 版本兼容性

- [ ] 在当前最新版 Nexus 中正常工作
- [ ] 没有使用已废弃的字段

## 提交 Bug 反馈

如果确定是 Nexus 的问题而不是拓展包的问题，请在 GitHub Issues 中反馈，并提供以下信息：

### 必选信息

- **Nexus 版本号**：设置页底部可以看到
- **Android 版本**：手机系统版本
- **问题描述**：清晰描述遇到的问题

### 可选但有帮助

- **拓展包 JSON**：如果方便，附上有问题的 JSON（可以脱敏）
- **截图/录屏**：可视化的问题更容易定位
- **复现步骤**：一步步说明如何复现问题

### Issue 模板

```
### 问题描述
清晰描述遇到的问题

### 复现步骤
1. 打开拓展包管理
2. 导入 xxx.json
3. 点击启用
4. 出现 xxx 问题

### 预期行为
描述你期望的正确行为

### 环境信息
- Nexus 版本：x.x.x
- Android 版本：x.x
- 手机型号：xxx

### 附件
（截图、拓展包文件等）
```

## 自测 Checklist

发布前对照这个清单检查一遍：

- [ ] JSON 格式正确，能通过 jsonlint 校验
- [ ] 所有必填字段已填写
- [ ] ID 命名规范，使用了独特的命名空间
- [ ] 方块/物品/音效/粒子都能正常补全
- [ ] 快捷命令和模板正确显示
- [ ] 启用/禁用功能正常
- [ ] 卸载后完全移除
- [ ] 没有使用 `minecraft:` 前缀的自定义内容
- [ ] 描述信息准确无误
- [ ] 版本号符合语义化规范
